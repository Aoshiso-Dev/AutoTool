AutoTool コマンド詳細ガイド（配布版）

このファイルは、AutoTool の各コマンドを「何に使うか」「どこを設定すればよいか」がすぐ分かるようにまとめた説明書です。

==================================================
1. まず覚えるポイント
==================================================
- コマンドは上から順に実行されます。
- 条件（If）やループは、開始と終了をセットで使います。
- 画面対象を限定したい場合は「ウィンドウタイトル」「ウィンドウクラス名」を設定します。
- OCR/AI/画像検索は、環境や画面状態で結果が変わるため、しきい値・タイムアウト調整が重要です。
- 実行時は「実行前チェック」が走り、画像パス/OCR(tessdata)/プログラムパスの不備があると実行を開始しません。

==================================================
2. クリック操作カテゴリ
==================================================

[クリック]
- 用途: 指定座標をクリックします。
- 主な設定:
  - 座標(X,Y)
  - マウスボタン（左/右など）
  - 押下維持時間(ms)
  - 注入方式（MouseEvent / SendInput）
  - 移動シミュレート（ON/OFF）
  - 対象ウィンドウ
- 実行結果: クリックできれば成功。

[画像クリック]
- 用途: 画像を探し、見つかった位置をクリックします。
- 主な設定:
  - 検索画像
  - 一致しきい値（高いほど厳密）
  - 強調検索色（必要時のみ）
  - タイムアウト / 検索間隔
  - クリック設定（ボタン、押下維持時間、注入方式）
  - 対象ウィンドウ
- 実行結果: 見つかってクリックできれば成功。見つからなければ失敗。

[画像クリック(AI検出)]
- 用途: ONNXモデルで対象を検出し、指定クラスをクリックします。
- 主な設定:
  - ONNXモデル
  - クラスID
  - 信頼度しきい値 / IoUしきい値
  - クリック設定（ボタン、押下維持時間、注入方式）
  - 対象ウィンドウ
- 実行結果: 指定クラスが検出できれば中心座標をクリックして成功。

[ホットキー]
- 用途: Ctrl/Alt/Shift + キーを送信します。
- 主な設定:
  - Ctrl / Alt / Shift
  - キー
  - 対象ウィンドウ
- 実行結果: キー送信できれば成功。

[待機]
- 用途: 指定時間だけ待機します。
- 主な設定:
  - 待機時間(ms)
- 実行結果: 待機完了で成功。

[画像待機]
- 用途: 画像が現れるまで待機します（クリックはしない）。
- 主な設定:
  - 検索画像
  - 一致しきい値
  - 強調検索色
  - タイムアウト / 検索間隔
  - 対象ウィンドウ
- 実行結果: 見つかれば成功、タイムアウトで失敗。

==================================================
3. 条件分岐カテゴリ
==================================================

[条件 - 画像存在判定]
- 用途: 画像がある場合のみ、対応する「条件 - 終了」までを実行します。
- 主な設定:
  - 検索画像
  - 一致しきい値
  - 強調検索色
  - 対象ウィンドウ

[条件 - 画像非存在判定]
- 用途: 画像がない場合のみ、対応する「条件 - 終了」までを実行します。
- 主な設定: 画像存在判定と同じ。

[条件 - 文字存在判定]
- 用途: OCRで文字を読み取り、文字が見つかる場合のみ分岐内を実行します。
- 主な設定:
  - 検索文字列
  - マッチ方式（Contains / Equals）
  - 大文字小文字区別
  - OCR領域（X,Y,幅,高さ）
  - OCR設定（言語、PSM、前処理、最小信頼度、文字種制限）
  - tessdataディレクトリ（必要時のみ）
  - 対象ウィンドウ

[条件 - 文字非存在判定]
- 用途: OCRで文字が見つからない場合のみ分岐内を実行します。
- 主な設定: 文字存在判定と同じ。

[条件 - 画像存在判定(AI検出)]
- 用途: AI検出で対象クラスが存在する場合のみ分岐内を実行します。
- 主な設定:
  - ONNXモデル
  - クラスID
  - 信頼度しきい値 / IoUしきい値
  - 対象ウィンドウ

[条件 - 画像非存在判定(AI検出)]
- 用途: AI検出で対象クラスが存在しない場合のみ分岐内を実行します。
- 主な設定: 画像存在判定(AI検出)と同じ。

[条件 - 変数比較]
- 用途: 変数値を比較し、条件に合う場合のみ分岐内を実行します。
- 主な設定:
  - 変数名
  - 演算子（==, !=, >, <, >=, <=）
  - 比較値

[条件 - 終了]
- 用途: 条件分岐ブロックの終端です。
- 注意: 単独では使わず、各 If コマンドとセットで使用してください。

==================================================
4. ループカテゴリ
==================================================

[ループ - 開始]
- 用途: ループブロックを開始します。
- 主な設定:
  - ループ回数（0 は無限ループ）

[ループ - 中断]
- 用途: 現在のループを途中で抜けます。
- 使いどころ: 条件成立時に早めに次処理へ進みたいとき。

[ループ - 終了]
- 用途: ループブロックの終端です。
- 注意: 「ループ - 開始」とセットで使用してください。

==================================================
5. 変数操作カテゴリ
==================================================

[変数設定]
- 用途: 文字列値を変数に代入します。
- 主な設定:
  - 変数名
  - 値

[変数設定 - 画像検索]
- 用途: 画像検索結果を変数に保存します。
- 主な設定:
  - 画像検索設定（画像、しきい値、色、タイムアウト、間隔）
  - 未検出で失敗（ON: 失敗 / OFF: 続行）
  - 結果変数名（found, x, y）
  - 対象ウィンドウ
- 実行結果:
  - found には true/false
  - 見つかった場合に x/y を保存

[変数設定 - 文字検索]
- 用途: OCRで文字検索し、判定結果や抽出文字を変数に保存します。
- 主な設定:
  - 検索文字列、マッチ方式、大文字小文字
  - OCR領域とOCR設定
  - 未検出で失敗（ON/OFF）
  - 結果変数名（found, text, confidence）
  - 対象ウィンドウ
- 実行結果:
  - found に true/false
  - text に抽出文字
  - confidence に信頼度

[変数設定(OCR)]
- 用途: OCR結果を変数に保存します。
- 主な設定:
  - 変数名
  - OCR領域
  - OCR設定（言語、PSM、前処理、文字種制限、最小信頼度）
  - tessdataディレクトリ
  - 対象ウィンドウ
- 実行結果:
  - 信頼度が最小値以上なら抽出文字を保存
  - 未満なら空文字を保存

[変数設定(AI検出)]
- 用途: AI検出結果の値を変数に保存します。
- 主な設定:
  - 変数名
  - 検出モード（Class / Count / X / Y / Width / Height）
  - ONNXモデル
  - 信頼度しきい値 / IoUしきい値
  - 対象ウィンドウ
- 実行結果:
  - 検出モードに応じた値を変数へ保存

==================================================
6. システム操作カテゴリ
==================================================

[プログラム実行]
- 用途: 外部プログラムを起動します。
- 主な設定:
  - プログラムパス
  - 引数
  - 作業ディレクトリ
  - 終了を待つ（ON/OFF）
- 実行結果: 起動できれば成功。エラー時はログに原因表示。

[スクリーンショット]
- 用途: 画面または対象ウィンドウを画像保存します。
- 主な設定:
  - 保存先ディレクトリ
  - 対象ウィンドウ
- 実行結果: PNG保存に成功すれば成功。
- 補足: 保存先未指定時は `Screenshots` フォルダが使われます。

==================================================
7. 設定のコツ（失敗しにくくする）
==================================================
- 画像検索が不安定な場合:
  - しきい値を少し下げる（例: 0.90 -> 0.80）
  - 検索間隔を短く、タイムアウトを長くする
- OCRが不安定な場合:
  - OCR領域を狭くして対象文字に寄せる
  - 前処理を Gray / Binarize で試す
  - 言語を jpn / jpn+eng で切り替える
- AI検出が不安定な場合:
  - 信頼度しきい値を下げる
  - クラスIDがモデル定義と一致しているか確認
- 条件分岐やループが動かない場合:
  - 開始と終了のペアが正しいか確認
