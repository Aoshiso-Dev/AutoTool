AutoTool ご利用ガイド（配布版）

このファイルは、配布 ZIP を受け取った利用者向けの説明です。

--------------------------------------------------
1. 使い始める前に
--------------------------------------------------
- AutoTool は Windows 用アプリです。
- この配布版はフレームワーク依存配布です。
  起動できない場合は .NET 10 Desktop Runtime のインストールが必要です。

--------------------------------------------------
2. 配布ファイルの置き方
--------------------------------------------------
- GitHub から取得した ZIP は、まず任意フォルダへすべて展開してください。
- `AutoTool.exe` と同じフォルダに、同梱 dll 一式をそのまま置いてください。
- フォルダ構成を変えたり、dll を一部だけ移動/削除しないでください。

推奨例:
  C:\AutoTool\
    AutoTool.exe
    *.dll
    *.deps.json / *.runtimeconfig.json
    Readme.txt
    Readme_コマンド詳細.txt
    THIRD_PARTY_NOTICES.txt
    licenses\*.txt
    Settings\appsettings.json

--------------------------------------------------
3. 起動方法
--------------------------------------------------
- `AutoTool.exe` をダブルクリックして起動します。
- コマンドライン引数でも起動・制御できます（同一PCで起動中の AutoTool に対しても有効）。
- タイトルバー右側の `バージョン情報` から、アプリ名/バージョン/GitHub URL を確認できます。
- GitHub URL はクリックしてブラウザで開けます。
- 正式バージョンの正は GitHub タグ（`vMAJOR.MINOR.PATCH`）です。

主な引数:
- `AutoTool.exe "C:\Macro\sample.macro"`
  - マクロを読み込んで起動します（実行はしません）。
- `-macro <path>`
  - マクロファイルを明示指定します。
- `-start`
  - マクロを開始します。`-macro` / ファイル指定と同時指定時は読込後に自動開始します。
- `-stop`
  - 実行中マクロの停止を要求します。
- `-exit`
  - アプリの終了を要求します（即時）。
- `-exit-on-complete`
  - マクロ完了後にアプリを自動終了します（`-start` 必須）。
- `-hide`
  - メインウィンドウを非表示にします。
- `-show`
  - メインウィンドウを表示して前面化します。
- `-silent-errors`
  - コマンドライン経由の実行失敗時に、警告/エラーダイアログを表示しません（ログは記録されます）。

引数の制約:
- `-start` と `-stop` は同時に指定できません。
- `-exit` と `-exit-on-complete` は同時に指定できません。
- `-hide` と `-show` は同時に指定できません。
- `-exit-on-complete` は `-start` と同時指定が必要です。

代表例:
- `AutoTool.exe "C:\Macro\sample.macro" -start -hide -exit-on-complete`
  - マクロを読込・開始し、非表示実行、完了後に自動終了します。
- `AutoTool.exe -stop -exit`
  - 実行停止を要求してから終了要求を送ります。

起動しない場合:
- SmartScreen が表示されたら「詳細情報」→「実行」を選択
- .NET 10 Desktop Runtime をインストール後、再度起動

--------------------------------------------------
4. 基本操作（最短）
--------------------------------------------------
1. コマンドを追加して手順を作成
   - `追加` ボタンからコマンド一覧ウィンドウを開き、説明文を確認しながら選択できます。
   - ループ/条件/リトライは開始を追加すると終了も自動で追加されます。
   - 開始または終了を削除すると、対応する相方も自動で削除されます。
2. 必要な値を設定
3. `.macro` として保存
4. 実行前チェック結果を確認（`診断` ボタンで表示切替）
5. 実行してログや変数を確認

緊急停止:
- 実行中に `Esc` を約 1.2 秒長押しすると、マクロを緊急停止できます。
- AutoTool が非アクティブな状態でも有効です。
- 実行中でも `停止` ボタンから停止できます。

実行前チェック:
- 実行ボタン押下時に、画像パス/OCR(tessdata)/外部実行ファイルパスを一括診断します。
- 要修正がある場合は実行せず、チェック一覧にエラーコード付きで表示します。
- 右下の `診断` ボタンでパネルを開閉できます。

変数確認:
- 右下の `変数` ボタンで、実行中に設定された変数名・値・更新時刻を確認できます。
- `ログ` と同じ下部領域に表示され、検索・コピー・更新ができます。

--------------------------------------------------
5. データ保存場所
--------------------------------------------------
- マクロ: `Macro\` または保存した `*.macro`
- 設定: `Settings\appsettings.json`
- 画面状態: `Settings\window_settings.json`

--------------------------------------------------
6. 更新版を上書き配布するときの注意
--------------------------------------------------
- 既存のユーザーデータを残すため、次は削除しないでください。
  - `Macro\`
  - `Settings\`
- 通常は `AutoTool.exe` と dll を新しいものへ差し替えてください。

--------------------------------------------------
8. よくあるトラブル
--------------------------------------------------
- 「dll が見つからない」と表示される:
  - 配布物一式を同じフォルダに展開し直してください。
- OCR が動かない/精度が低い:
  - 設定画面で tessdata ディレクトリを確認してください。
  - `jpn.traineddata` / `eng.traineddata` の有無を確認してください。
- 保存できない:
  - 配置先フォルダの書き込み権限を確認してください。
